Problem Definition

For this project, we need to ddevelop an algorithm to identify the pianist's hands from a sequence of video frames in which a person is playing the piano with both hands. Portions of the hands are sometimes in deep shadow, which creates a challenging imaging situation.

Method and Implementation

Firstly, we do a strict skin detection to make a binary image with only skin-color objects left white, which are mainly the piano, part of head and two hands. Then we do the vertical projection. There are gaps between blocks of white pixels. Between piano and body parts, the gap will be particularly large and we name the column as k. We just swipe all of the pixels left to k column to black. We do the same to head which is right to the both hands. When we are left with two hands, which are two connected components, we draw bounding box around them.

Functions:
Skin Detection-- void mySkinDetect(Mat& src, Mat& dst)
Projection and Process Pixels and Mark Hands -- void check(Mat& image, Mat& image0)

Discussion

Strengths: Our algorithm did well in all the frames
Limitations: There are still quite a bit limitations as our segementation is set according to this particular video. Only when the sky view is took and the light condition is good, we can detect the pianist' hands. Also, if the video is not about playing piano, due to the special segementation ways, the algorithm will not work.
Potential future works: If we can try other method so that the algorithm can apply to more situations

Conclusions

In conclusion, our algorithms works for all the frames, detecting both moving hands.
